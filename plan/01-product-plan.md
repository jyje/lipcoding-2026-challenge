# 01. Product Plan

## 1) 제품 정의

- 제품명: `ThriveOps`
- 목적: 개인 업무, 커리어, 테크 지식 입력을 통합 분석해 "지금 무엇을 해야 하는지"를 제안
- 포지셔닝: 기록 도구가 아니라 실행 의사결정 도구
- 진입 가정: 앱을 처음 실행하면 "하루의 업무 시작" 시점으로 가정하고, 오늘 하루를 여는 초기 화면을 보여준다

## 2) 문제 정의

사용자는 아래 문제를 동시에 겪습니다.

- 업무: 급한 일과 중요한 일을 구분하기 어려움
- 커리어: 장기 목표와 오늘 행동이 연결되지 않음
- 지식: 배운 내용을 실무 행동으로 전환하지 못함

## 3) 해결 가설

자유 입력(브레인덤프)만 받아도 AI가 다음을 생성하면 실행률이 올라간다.

- 오늘의 Top 3 행동
- 이번 주 커리어 기여 행동
- 30분 단위 실행 플랜

## 4) 대상 사용자

- 직장인/개발자/취준생 등 멀티 목표를 동시에 관리하는 개인
- 데일리 계획은 세우지만 지속 실행이 어려운 사용자

## 5) 핵심 사용자 스토리

1. 사용자로서, 내가 떠오르는 일을 한 번에 입력하면 우선순위가 정리되길 원한다.
2. 사용자로서, 커리어 목표 대비 오늘의 행동이 무엇인지 알고 싶다.
3. 사용자로서, 학습 내용을 실제 업무 태스크로 변환받고 싶다.
4. 사용자로서, 남은 시간 기준으로 당장 실행 가능한 플랜을 받고 싶다.

## 6) MVP 범위 (대회 3시간 기준)

Must

- 브레인덤프 입력 1개
- 카테고리 태깅: `work | career | tech`
- AI 분석 결과: Top 3 + 이유 + 30분 플랜
- 액션 체크(완료 토글)
- Azure 배포 URL에서 동작

Should

- 재분석 버튼(남은 시간 반영)
- 리스크 경고 1개(예: 마감 임박)

Could

- 주간 리포트
- 간단 점수 차트

## 7) 비기능 목표

- 응답 시간: 분석 API p95 8초 이하
- 실패 대응: AI 구조화 실패 시 fallback 응답 제공
- 로그: 요청 ID, 지연시간, 실패 원인 기록

## 8) 성공 지표 (대회용)

- 엔드 투 엔드 성공률 100% (데모 시)
- 초기 입력 후 1회 클릭으로 결과 화면 도달
- 심사 Q&A에서 Copilot SDK 사용 목적을 명확히 설명 가능

## 9) 백엔드 핵심 기능 (개발자 커리어 루프)

일상 개발 업무와 커리어 성장을 하나의 자동 루프로 연결하는 백엔드 핵심 기능입니다.

### 9-1) 채용 공고 기반 역량 분석

- 사전 설정된 약 10개 회사의 채용 공고를 분석한다.
- 공고에서 개발자에게 요구되는 업무 역량(스킬/키워드)을 추출·정리한다.
- 목적: 일상 개발 업무를 수행하면서 동시에 **커리어 발전 방향**을 함께 잡는다.

### 9-2) 지식기반 기반 문제 해결 + 축적

- 일상 업무에서 이미 구축된 **지식기반(knowledge base)** 을 활용해 문제를 해결한다.
- 해결 과정에서 새로 알게 된 내용을 다시 지식기반에 **지식 항목으로 축적**한다.
- 지식기반은 시간이 지날수록 풍부해지는 누적형(compounding) 자산이 된다.

### 9-3) 자동 반복 학습 루프

- 위 두 과정(역량 분석 → 문제 해결 → 지식 축적)을 자동으로 반복한다.
- 반복을 통해 **지속적인 학습**을 생성하고, 커리어 역량을 함께 발전시킨다.
- 한 줄 요약: 매일의 개발 업무가 곧 커리어 성장 데이터로 누적되는 선순환.

## 10) 제외 범위

- 외부 캘린더 OAuth 연동
- 복잡한 권한 체계
- 실시간 협업 기능
