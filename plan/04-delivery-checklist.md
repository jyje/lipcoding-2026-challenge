# 04. Delivery Checklist

## 1) 기능 체크

- [ ] 브레인덤프 입력 후 분석 결과 생성
- [ ] Top 3 액션과 이유 표시
- [ ] 30분 실행 플랜 표시
- [ ] 액션 완료 토글 가능
- [ ] 재계획 버튼 동작

## 2) 안정성 체크

- [ ] 빈 입력 처리
- [ ] 과도한 길이 입력 제한
- [ ] AI 스키마 실패 fallback 동작
- [ ] 서버 에러 메시지 사용자 친화 출력

## 3) Copilot SDK 체크

- [ ] SDK 실제 호출 경로 데모 가능
- [ ] 모델 응답 스키마 검증 적용
- [ ] 핵심 프롬프트/템플릿 설명 가능

## 4) Azure 체크

- [ ] BE 배포 URL 헬스체크 OK
- [ ] FE 배포 URL 렌더링 OK
- [ ] FE -> BE CORS 정상
- [ ] 환경변수 누락 없음

## 5) 데모 체크

- [ ] 데모용 입력 샘플 2개 준비
- [ ] 60초 대본 2회 리허설 완료
- [ ] 네트워크 장애 대비 백업 시나리오 준비

## 6) 제출 직전 체크

- [ ] README에 실행 방법/아키텍처/한계 작성
- [ ] 오픈소스 라이선스 확인
- [ ] 하드코딩 시크릿 제거

## 7) 리스크 대응표

| 리스크 | 징후 | 대응 |
|---|---|---|
| SDK 응답 불안정 | JSON 파싱 실패 | 재시도 1회 + fallback |
| Azure 배포 지연 | 빌드 실패/환경변수 오류 | 로컬 데모 병행 + 최소 수정 |
| UI 완성도 부족 | 화면 복잡/가독성 저하 | 카드 3개 중심으로 단순화 |
| 시간 초과 | 통합 버그 누적 | Gate 기준으로 기능 컷 |
