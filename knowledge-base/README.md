## 🎯 사용 방법

### 정보 찾기
1. 위의 카테고리에서 관련 섹션을 찾습니다
2. 각 섹션의 README에서 원하는 주제를 검색합니다
3. 문서를 읽고 필요시 관련 링크를 따릅니다

### 정보 추가 및 수정
1. 해당 카테고리 폴더에서 파일을 생성하거나 수정합니다
2. Markdown 포맷을 사용합니다 (지원하는 경우 예시, 다이어그램 포함)
3. 명확한 제목과 설명을 사용합니다
4. 코드 예제가 있으면 최신으로 유지합니다

### 에이전트 참조
에이전트 시스템은 다음 경로를 자동으로 참조합니다:
- `.github/instructions/` - 에이전트별 가이드
- `knowledge-base/` - 팀 전체 지식
- `deployment/` - 배포 인프라 정보

## 📌 핵심 원칙

### 1. 단일 정보원
- 같은 정보를 여러 곳에 중복하지 않습니다
- 최신 정보는 여기에만 보관합니다

### 2. 명확성
- 간결하고 실행 가능한 정보를 제공합니다
- 복잡한 개념은 다이어그램이나 예제로 설명합니다

### 3. 접근성
- 모든 팀 멤버가 읽을 수 있어야 합니다
- 기술 용어는 정의하거나 용어집을 참조합니다

### 4. 유지보수
- 정기적으로 최신 상태 확인
- 변경사항이 있으면 즉시 업데이트

### 5. 협업
- 모든 팀 멤버와 에이전트가 기여할 수 있습니다
- 피드백과 개선을 환영합니다
