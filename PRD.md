# ThriveOps PRD

## 1. 제품 개요

ThriveOps는 업무, 커리어, 테크 지식을 한 화면에 모아 "지금 무엇을 해야 하는지"를 제안하는 실행 중심 워크스페이스다. 단순 기록 도구가 아니라, 입력을 실행 계획과 다음 행동으로 바꾸는 데 초점을 둔다.

한 줄 요약: 사용자가 흩어 놓은 생각을 AI가 정리하고, 오늘 바로 실행할 수 있는 작업으로 바꿔준다.

## 2. 문제 정의

사용자는 보통 세 가지 축을 동시에 다룬다.

- 당장 처리해야 할 업무
- 커리어 성장을 위해 쌓아야 하는 행동
- 학습했지만 실행으로 연결되지 않는 지식

기존 도구는 이 셋을 따로 관리하게 만들기 때문에, 오늘 해야 할 일의 우선순위와 장기 성장 방향이 분리된다.

## 3. 목표

- 입력을 한 번에 받아 실행 가능한 작업으로 전환하기
- 업무와 커리어를 분리하지 않고 같은 흐름에서 다루기
- AI가 요약, 분류, 다음 행동 추천을 담당하게 하기
- 대화형 입력으로 수정/추가/질문을 자연스럽게 처리하기

## 4. 핵심 사용자

- 직장인, 개발자, 취업 준비생처럼 목표가 여러 개인 사용자
- 계획은 세우지만 실행 전환이 느린 사용자
- 업무와 커리어를 동시에 관리해야 하는 사용자

## 5. 사용자 스토리

### 5.1 입력과 정리

사용자로서, 떠오르는 일을 한 번에 입력하면 AI가 업무/커리어/테크로 정리해 주길 원한다.

### 5.2 우선순위 판단

사용자로서, 지금 가장 먼저 해야 할 일을 알고 싶다.

### 5.3 대화형 수정

사용자로서, 말로 "내일 일정으로 바꿔줘", "이건 면접 준비로 옮겨줘"처럼 수정하고 싶다.

### 5.4 커리어 연결

사용자로서, 오늘의 업무가 내 커리어 성장과 어떤 관계가 있는지 보고 싶다.

### 5.5 실행 전환

사용자로서, 추천만 받는 데서 끝나지 않고 바로 체크/수정/추적이 되길 원한다.

## 6. 기능 범위

### 6.1 Must

- 브레인덤프/메시지 입력
- 업무/커리어/테크 분류
- AI 요약과 Top action 추천
- 하단 대화창에서 등록, URL 붙여넣기, 파일 드롭, 명령형 입력 지원
- 카드 상태 변경 및 날짜 수정
- 기본 인증과 토큰 기반 API 호출

### 6.2 Should

- 대화형 수정/삭제/상태 변경
- 최근 컨텍스트를 바탕으로 한 재분석
- 접근성 보강: 툴팁, 설명 버튼, live region

### 6.3 Could

- 지식기반 축적
- 채용 공고 역량 집계
- 주간 리포트
- 반복 학습 루프

## 7. 현재 구현 상태와 현실성 평가

### 이미 구현된 것

- Next.js 프런트엔드와 Fastify 기반 백엔드의 기본 골격
- 대화창 UI와 접근성 보강 일부
- AI 분석 API와 대화형 intent 처리 흐름
- 업무/커리어/테크 항목 표시와 카드 편집 흐름
- Docker 및 배포 문서 정리
- 루트 README와 기획/아키텍처 문서 정리

### 구현 의도가 분명했지만 아직 문서/계획 중심인 것

- 지식기반의 누적형 문제 해결 루프
- 채용 공고 기반 역량 집계
- 대화 이력/변경 이력의 완전한 추적 로그
- 사용자별 컨텍스트를 강하게 분리한 다중 세션 운영

### 비판적 평가

- 강점: 제품 메시지는 분명하다. "기록"이 아니라 "실행"이라는 포지셔닝이 흔들리지 않고, 업무와 커리어를 함께 다루는 점이 차별화된다.
- 강점: 하단 대화창은 단순 입력 박스보다 훨씬 낫다. 등록, 파일 드롭, URL, 명령형 입력이 한 흐름에 묶여 있어 데모 설명력이 좋다.
- 강점: 문서, 배포, 구조화된 AI 응답까지 갖춰져 있어 구현 방향 자체는 일관적이다.
- 약점: 일부 고급 아이디어는 현재 구현보다 기획이 앞서 있다. 특히 지식기반 루프와 역량 집계는 데이터가 충분히 쌓이기 전까지는 설득력이 약하다.
- 약점: AI가 실제로 무엇을 바꾸는지에 대한 증거가 부족하면 "멋진 데모"로 보일 수 있다. 따라서 변경 이력과 결과 비교 화면이 중요하다.
- 약점: 기능이 많아질수록 사용자가 "이 앱에서 첫 행동이 무엇인지" 헷갈릴 수 있다. 초반에는 핵심 플로우를 더 강하게 좁혀야 한다.

## 8. 핵심 플로우

### 플로우 A: 오늘 시작하기

1. 사용자가 업무/커리어/테크 내용을 입력한다.
2. AI가 요약과 분류를 만든다.
3. 사용자는 추천 액션을 확인한다.
4. 카드를 수정하거나 완료 처리한다.

### 플로우 B: 대화로 수정하기

1. 사용자가 하단 대화창에 명령을 입력한다.
2. 시스템이 intent를 추론한다.
3. 대상 항목을 수정하고 변경 결과를 돌려준다.
4. 결과는 화면에 남고, 필요하면 인사이트로 정리된다.

### 플로우 C: 커리어 연결 보기

1. 사용자가 커리어 관련 항목을 입력한다.
2. 시스템이 관련 기술/키워드를 태깅한다.
3. 사용자는 현재 업무와 커리어의 연결을 확인한다.

## 9. 데이터와 운영 원칙

- 모든 핵심 변경은 추적 가능해야 한다.
- AI 결과는 구조화된 형태로 받아야 한다.
- 실패 시에는 fallback이 동작해야 한다.
- 사용자 경험은 화려함보다 실패하지 않는 흐름을 우선한다.

## 10. 성공 기준

- 데모 상황에서 엔드 투 엔드 흐름이 끊기지 않는다.
- 사용자가 1회 입력으로 AI 결과와 액션을 확인할 수 있다.
- 대화창만 보고도 무엇을 할 수 있는지 이해된다.
- 제품이 "업무 도구"가 아니라 "실행 도구"로 기억된다.

## 11. 리스크

- 기능 범위가 넓어지면 핵심 메시지가 흐려질 수 있다.
- AI가 매끄럽지 않으면 제품 가치가 약해 보일 수 있다.
- 계획 대비 구현이 부족한 고급 기능은 과장으로 읽힐 수 있다.

## 12. 현재 우선순위 제안

1. 대화창과 카드 수정 플로우를 안정화한다.
2. 변경 이력과 결과 피드백을 더 선명하게 보여준다.
3. 아직 문서 수준인 커리어/지식 루프는 작은 범위부터 단계적으로 붙인다.
